Parses the nano_core object file that represents the already loaded (and currently running) nano_core code. Basically, just searches for global (public) symbols, which are added to the system map and the crate metadata.

Return

If successful, this returns a tuple of the following:

  • nano_core_crate_ref: A reference to the newly-created nano_core crate.
  • init_symbols: a map of symbol name to its constant value, which contains assembler and linker constances.
  • The number of new symbols added to the symbol map (a usize).

If an error occurs, the returned Result::Err contains the passed-in text_pages, rodata_pages, and data_pages because those cannot be dropped, as they hold the currently-running code, and dropping them would cause endless exceptions.
