pub fn pci_enable_msi(
    &self,
    core_id: u8,
    int_num: u8
) -> Result<(), &'static str>[src]

Enable MSI interrupts for a PCI device. We assume the device only supports one MSI vector and set the interrupt number and core id for that vector. If the MSI capability is not supported then an error message is returned.

Arguments

  • core_id: core that interrupt will be routed to
  • int_num: interrupt number to assign to the MSI vector

pub fn pci_enable_msix(&self) -> Result<(), &'static str>[src]

Enable MSI-X interrupts for a PCI device. Only the enable bit is set and the remaining initialization steps of setting the interrupt number and core id should be completed in the device driver.

pub fn find_pci_capability(&self, pci_capability: u16) -> Option<u16>[src]

Explores the PCI config space and returns address of requested capability, if present. PCI capabilities are stored as a linked list in the PCI config space, with each capability storing the pointer to the next capability right after its ID. The function returns a None value if capabilities are not valid for this device or if the requested capability is not present.
